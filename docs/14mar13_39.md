## Pages wired

### AdminCampaignList.tsx
- `useQuery` fetches `campaigns` joined with `clients.name` and `profiles.display_name` (driver)
- Ordered by `campaign_date DESC`
- Client-side search filter across title, client name, driver name
- Loading spinner + empty state + error state added

### DriverCampaign.tsx
- Fetches today's assigned campaign for the logged-in driver (first campaign `>= today`)
- **Start Shift**: `useMutation` inserts into `driver_shifts` with `started_at = now()`
- **End Shift**: `useMutation` updates `ended_at` on the open shift, then navigates to `/driver/login`
- Shows real shift start time when active
- Recent uploads list from real `campaign_photos` rows (sorted by `submitted_at DESC`, capped at 5)
- Handles "no campaign assigned" state gracefully

### DriverUpload.tsx
- Two separate hidden `<input type="file">` elements: one with `capture="environment"` (camera), one without (gallery)
- Object URL preview of selected image
- On submit: uploads file to `campaign-photos` storage bucket at path `campaigns/{campaignId}/photos/{uuid}/original.{ext}`
- If storage upload succeeds, inserts `campaign_photos` row; if DB insert fails, removes the orphaned storage object
- Navigates to `/driver/upload-success` on completion
- Campaign ID sourced from TanStack Query cache (same query key as DriverCampaign â€” no extra network call)

### AdminPhotoApproval.tsx
- Fetches all `campaign_photos` for the campaign from URL param `:id`
- Fetches campaign title separately for the header
- Generates signed URLs for all photos (1-hour expiry) via `supabase.storage.createSignedUrl`
- **Approve / Reject** per photo: updates `status`, `reviewed_by`, `reviewed_at`
- **Approve All**: runs all pending approvals in parallel, shows toast with count
- Mobile "Approve All" button shown below grid
- Invalidates `["campaign", id]` query so AdminCampaignDetail badge counts refresh

### ClientCampaignView.tsx
- Fetches most recent campaign for the client's `client_id` (from profile)
- Photos fetched via `campaign_photos` â€” RLS policy `campaign_photos: client read approved` ensures only `status = 'approved'` rows are returned
- Signed URLs fetched for all approved photos
- Loading skeleton while URLs resolve; graceful "no campaign" state

### ClientTimingSheet.tsx
- Fetches most recent campaign for client, then its first `driver_shifts` row, then first approved photo `submitted_at`
- Timeline rows: Shift Start, First Photo, Shift End â€” each shows real time or "â€”"
- CheckCircle icon for completed events, Clock icon for pending

- Client-side validation: title, client, date required
- On success: invalidates campaign list cache, navigates to new campaign detail
- Cost fields: `driver_daily_wage`, `transport_cost`, `other_cost` with step="0.01"
